## Data Life Cycle

The data life cycle refers to the entire journey of data — from the time it is created or collected until it is deleted.

### Six Stages of the Data Life Cycle

**1. Plan**  
In this stage, a company decides what type of data it needs, why it needs it, who will manage it, and what outcomes it expects. Planning ensures that data collection has a clear purpose.

**2. Capture**  
Data is collected from different sources such as surveys, transactions, systems, or online platforms and brought into the organization.

**3. Manage**  
This stage focuses on storing, protecting, and maintaining data. It includes deciding where the data will be stored, how it will be secured, and how its quality will be maintained.

**4. Analyze**  
Here, data is examined to solve problems and support business goals. Analysts may use spreadsheets, formulas, reports, or other tools to understand patterns and trends.

**5. Archive**  
Important data is stored safely for long-term use or future reference.

**6. Destroy**  
Data that is no longer needed is permanently deleted. This protects sensitive company and customer information.

Although different companies may structure these stages differently, one rule always applies:  
Data must be accurate, secure, and accessible when needed.

Stakeholders are individuals or groups who are involved in a project or affected by its results.  
They invest time, effort, or resources and care about the outcome.

## Data Analyst Tools

### Spreadsheets

A spreadsheet is a digital tool used to store and organize data in rows and columns.  
Common examples include Microsoft Excel and Google Sheets.

Spreadsheets help analysts:
- Organize and sort data  
- Identify patterns  
- Perform calculations  
- Create charts and graphs  

**Formula**  
A formula is a custom calculation created by the user.

**Function**  
A function is a built-in command that performs a specific calculation automatically.

---

### Query Language

A query language allows analysts to retrieve and manage data stored in databases.

**SQL (Structured Query Language)** is used to communicate with databases.  
A database is an organized collection of data stored in a computer system.

With SQL, analysts can:
- Retrieve specific information  
- Filter and sort data  
- Add or modify records  
- Download data for analysis  

Common SQL platforms include MySQL, Microsoft SQL Server, and BigQuery.

---

### Data Visualization

Data visualization is the process of presenting data visually using charts, graphs, maps, or dashboards.

Visualization tools help turn complex numbers into insights that are easy to understand.

Popular tools include Tableau and Looker.

These tools:
- Make data easier to interpret  
- Help stakeholders make informed decisions  
- Allow creation of interactive dashboards  

For example:
- Tableau allows drag-and-drop dashboard creation.  
- Looker connects directly to databases for real-time data access.

---

### Programming Languages

Data analysts may also use programming languages like R and Python.

These are commonly used for:
- Statistical analysis  
- Data cleaning  
- Automation  
- Advanced visualization

## Data Life Cycle vs Data Analysis Process

The data life cycle focuses on how data is collected, stored, managed, and maintained over time.

The data analysis process focuses on how data is used to answer questions and solve business problems.

### Similarities
- Both involve structured steps.
- Both support business goals.
- Both require planning and organization.

### Differences
- The data life cycle is about managing data.
- The data analysis process is about interpreting and using data.

Good data management makes good analysis possible.

---

## Ask Phase vs Plan Phase

Both the Ask phase and Plan phase involve preparation, but they focus on different things.

**Ask Phase (Data Analysis Process)**  
- Defines the business problem  
- Sets goals  
- Identifies key questions  

**Plan Phase (Data Life Cycle)**  
- Identifies what data is needed  
- Determines where to get the data  
- Decides how the data will be managed  

In simple terms:  
Ask focuses on business questions.  
Plan focuses on data preparation.
