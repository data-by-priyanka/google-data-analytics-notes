## Database

A **database** is a structured collection of data stored in a computer system.

Databases allow users to:
- Store large amounts of information
- Retrieve data efficiently
- Keep information organized and consistent

---

## Relational Database

A **relational database** organizes data into related tables.

Tables are connected through shared fields.  
To create a relationship, at least one column must exist in both tables.

Relational databases:
- Keep data consistent
- Reduce duplication
- Make analysis easier
- Allow multiple users to access the same information reliably

In contrast, non-relational structures often store everything in one large group, which makes analysis harder.

---

## Keys in Relational Databases

### Primary Key

A **primary key**:
- Uniquely identifies each record in a table
- Cannot contain null values
- Must contain unique values
- Only one primary key is allowed per table

A **composite key** is a primary key made up of multiple columns.

---

### Foreign Key

A **foreign key**:
- Connects one table to another
- References the primary key in another table
- Can appear multiple times in a table

Foreign keys create relationships between tables.

---

# Metadata

**Metadata** is data about data.

It explains:
- Who created the data
- What the data represents
- When it was collected
- Where it came from
- How it was collected
- Why it was created

Metadata helps analysts:
- Understand data context
- Combine datasets
- Evaluate data quality
- Reuse data effectively

---

## Types of Metadata

**Descriptive Metadata**  
Describes the content of the data so it can be identified later.

**Structural Metadata**  
Explains how data is organized and related.

**Administrative Metadata**  
Provides technical details about how and where data is stored.

---

## Metadata Repository

A **metadata repository** is a centralized system that stores metadata.

It helps organizations:
- Track where data is stored
- Understand how systems are connected
- Monitor data flow
- Ensure consistency
- Track data access

A centralized metadata system creates a single source of truth.

---

# Data Governance

**Data governance** is the formal management of a company’s data assets.

It ensures:
- Data security
- Data privacy
- Data integrity
- Proper data usage
- Controlled access

Strong data governance improves reliability and compliance.

---

## Metadata Specialists

Metadata specialists:
- Organize company data
- Maintain data quality
- Define data standards
- Document how systems connect

They ensure consistency across the organization.

---

# CSV Files

**CSV (Comma-Separated Values)** files store data in plain text table format.

Key characteristics:
- Values are separated by delimiters (usually commas)
- Easy to import into spreadsheets
- Useful for sharing data across systems

CSV files help analysts:
- Examine small portions of large datasets
- Move data between tools
- Separate values clearly

---

# Importing Data in Spreadsheets

Common methods include:

- Importing from other spreadsheets
- Importing CSV files
- Importing HTML tables from web pages

Spreadsheet tools provide built-in functions for importing external data.

---

# Sorting and Filtering

## Sorting

Sorting arranges data in a specific order (ascending or descending) to make it easier to analyze.

---

## Filtering

Filtering shows only the data that meets specific criteria.

It hides irrelevant data and simplifies analysis.

---

# BigQuery

BigQuery is a cloud-based data warehouse that allows analysts to:

- Query very large datasets
- Filter and aggregate data
- Perform complex SQL operations
- Store and analyze data at scale

It is commonly used for large-scale analytics.

---

## Key Takeaway

Understanding databases and metadata allows analysts to:

- Work with structured systems
- Maintain data integrity
- Connect related information
- Manage large datasets efficiently
- Ensure data is reliable and well-governed

Strong database knowledge turns raw data into organized, usable information.
