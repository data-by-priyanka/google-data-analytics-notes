## How Data Is Collected

Data can be collected in many ways, depending on the purpose of the analysis.

Common methods include:
- Interviews
- Observations (often used in scientific research)
- Forms and questionnaires
- Surveys
- Website cookies

Each method has strengths and weaknesses. The key is choosing the method that best fits your business question.

---

## Data Collection Considerations

Before collecting data, analysts must think carefully about the process.

### 1. How Will the Data Be Collected?

You can:
- Collect data yourself
- Purchase or receive data from another source

If collecting data internally, you have more control over quality.

---

### 2. Choose the Data Source

**First-party data**  
Collected directly by your organization using its own resources.  
Usually the most reliable.

**Second-party data**  
Collected by another organization and shared or sold directly to you.

**Third-party data**  
Collected by external sources and often aggregated from multiple places.  
May require extra verification for reliability.

No matter the source, always check for:
- Accuracy
- Bias
- Credibility

---

### 3. Decide What Data to Use

Focus only on data that helps answer your specific business question.  
Avoid distractions from unrelated data.

---

### 4. How Much Data to Collect?

**Population**  
All possible data values in a dataset.

**Sample**  
A smaller group taken from the population that represents it.

When studying an entire population isn’t practical, analysts use samples.

---

### 5. Select the Right Data Type

Different questions require different types of data (quantitative, qualitative, continuous, etc.).

---

### 6. Determine the Time Frame

Decide:
- How long to collect data
- Whether historical data is sufficient
- Whether trend analysis is required

Time frame affects the accuracy and relevance of your conclusions.

# Data Formats and Types

## Primary vs Secondary Data

**Primary Data**  
Collected firsthand by the researcher.

Examples:
- Surveys you conducted
- Interviews you performed

**Secondary Data**  
Collected by someone else.

Examples:
- Census data
- Purchased customer data

---

## Internal vs External Data

**Internal Data**  
Exists within the organization.

Examples:
- Sales records
- HR data

**External Data**  
Comes from outside sources.

Examples:
- Industry reports
- Market statistics

---

## Continuous vs Discrete Data

**Continuous Data**  
Measured data with many possible values.

Examples:
- Temperature
- Height

**Discrete Data**  
Counted data with limited values.

Examples:
- Number of customers
- Tickets sold

---

## Qualitative vs Quantitative Data

**Qualitative Data**  
Describes characteristics or opinions.

Examples:
- Customer feedback
- Brand preferences

**Quantitative Data**  
Numeric and measurable.

Examples:
- Revenue
- Percentage growth

---

## Nominal vs Ordinal Data

**Nominal Data**  
Categories without order.

Examples:
- Customer type
- Job role

**Ordinal Data**  
Categories with ranking.

Examples:
- Star ratings
- Income level categories

---

## Structured vs Unstructured Data

**Structured Data**  
Organized in rows and columns.

Examples:
- Databases
- Spreadsheets

**Unstructured Data**  
Not organized in a predefined format.

Examples:
- Emails
- Social media posts
- Videos

---

# Data Modeling

Data modeling is the process of visually organizing data and showing how different data elements relate to each other.

Think of it like a blueprint.  
It helps everyone understand how data is structured.

**Data elements** are individual pieces of information, such as:
- Names
- Account numbers
- Addresses

Data models:
- Keep data consistent
- Show relationships
- Improve collaboration

---

## Levels of Data Modeling

### 1. Conceptual Model
High-level overview of how data connects across the organization.  
No technical details.

### 2. Logical Model
Defines relationships and attributes.  
Shows how records are uniquely identified.

### 3. Physical Model
Includes technical details like:
- Table names
- Column names
- Data types

---

## Data Modeling Techniques

Common methods include:
- Entity Relationship Diagrams (ERD)
- Unified Modeling Language (UML)

Most organizations already use a specific modeling method.

---

# Data Types

A data type defines what kind of value is stored.

In spreadsheets:
- Number
- Text (String)
- Boolean (True/False)

In SQL, data types vary by database system.

---

## Data Tables

A data table consists of:

- **Records** (rows)
- **Fields** (columns)

Records and fields are general database terms.  
Rows and columns are more commonly used in spreadsheets.

---

# Wide vs Long Data

## Wide Data

Each subject has one row with multiple columns.

Best for:
- Simple comparisons
- Basic charts

---

## Long Data

Each subject appears in multiple rows over time.

Best for:
- Time-based analysis
- Advanced statistical modeling

---

# Data Transformation

Data transformation means changing data’s format, structure, or values.

Common transformations include:
- Adding or deleting fields
- Renaming columns
- Merging datasets
- Standardizing formats
- Saving files in new formats (e.g., CSV)

---

## Goals of Data Transformation

- Better organization
- Improved compatibility
- Easier migration
- Data merging
- Enhanced detail
- Accurate comparisons

---

## Key Takeaway

Good data analysis starts with:
- Understanding how data is collected
- Choosing reliable sources
- Selecting appropriate data types
- Organizing data properly
- Transforming data when needed

Strong preparation leads to strong analysis.
